## API Authentication

### Authentication Methods

NUBI supports multiple authentication mechanisms for different use cases:

<Tabs>
  <TabItem label="JWT Bearer Token">
```bash
# Obtain authentication token
curl -X POST https://api.anubis.chat/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "your-username",
    "password": "your-password",
    "platform": "api"
  }'

# Response
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresAt": "2024-01-01T12:00:00Z",
  "refreshToken": "refresh-token-here",
  "user": {
    "id": "user_456",
    "username": "your-username",
    "permissions": ["raids:read", "raids:write", "leaderboards:read"]
  }
}

# Use token in subsequent requests
curl -X GET https://api.anubis.chat/api/v1/raids \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```
  </TabItem>
  
  <TabItem label="API Key">
```bash
# Generate API key (requires JWT authentication)
curl -X POST https://api.anubis.chat/api/v1/auth/api-keys \
  -H "Authorization: Bearer your-jwt-token" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Application",
    "permissions": ["raids:read", "leaderboards:read"],
    "expiresAt": "2024-12-31T23:59:59Z"
  }'

# Response
{
  "apiKey": "nubi_ak_1234567890abcdef",
  "keyId": "key_789",
  "name": "My Application",
  "permissions": ["raids:read", "leaderboards:read"],
  "createdAt": "2024-01-01T00:00:00Z",
  "expiresAt": "2024-12-31T23:59:59Z"
}

# Use API key in requests
curl -X GET https://api.anubis.chat/api/v1/raids \
  -H "X-API-Key: nubi_ak_1234567890abcdef"
```
  </TabItem>

## Error Responses

### Standard Error Format

All API errors follow a consistent format for easy handling:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "One or more fields failed validation",
    "details": {
      "field": "target.url",
      "reason": "Invalid URL format",
      "provided": "not-a-url",
      "expected": "Valid HTTP/HTTPS URL"
    },
    "timestamp": "2024-01-01T12:00:00Z",
    "requestId": "req_abc123456",
    "documentation": "https://docs.anubis.chat/api/errors#validation-error"
  }
}
```

### Common Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `AUTHENTICATION_REQUIRED` | 401 | No authentication token provided |
| `INVALID_TOKEN` | 401 | Authentication token is invalid or expired |
| `INSUFFICIENT_PERMISSIONS` | 403 | User lacks required permissions |
| `RESOURCE_NOT_FOUND` | 404 | Requested resource does not exist |
| `VALIDATION_ERROR` | 400 | Request data validation failed |
| `RATE_LIMITED` | 429 | Too many requests in time window |
| `INTERNAL_ERROR` | 500 | Unexpected server error |
| `SERVICE_UNAVAILABLE` | 503 | Service temporarily unavailable |

NUBI's Service APIs provide comprehensive programmatic access to all platform capabilities, enabling developers to build powerful integrations and custom applications on top of the NUBI ecosystem.
