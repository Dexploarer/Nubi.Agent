## üöÄ 5-Minute Setup

### Step 1: Clone & Install

<Tabs>
  <TabItem label="With Bun (Recommended)">
```bash
# Clone the repository
git clone https://github.com/anubis-chat/nubi.git
cd nubi

# Install dependencies (fast!)

bun install

# Verify installation

bun run type-check

````
  </TabItem>

  <TabItem label="With npm">
```bash
# Clone the repository
git clone https://github.com/anubis-chat/nubi.git
cd nubi

# Install dependencies
npm install

# Verify installation
npm run type-check
````

  </TabItem>
</Tabs>

### Step 2: Environment Setup

<Tabs>
  <TabItem label="Quick Setup (Minimal)">
```bash

cp .env.example .env

# Edit with your preferred editor

code .env # or nano .env, vim .env

# Minimal required configuration:

NODE_ENV=development
OPENAI_API_KEY=your-openai-key-here
PGLITE_DATA_DIR=./.eliza/.elizadb

````
  </TabItem>

  <TabItem label="Full Setup (Recommended)">
```bash

# AI Model (choose one)
OPENAI_API_KEY=sk-your-openai-key
# OR

# Twitter Integration (optional)
TWITTER_API_KEY=your-twitter-key
````

  </TabItem>
</Tabs>

## üêõ Troubleshooting

<Aside type="caution">
**Common Issues & Solutions**
</Aside>

### Database Issues
```bash
# Clear PGLite database
rm -rf ./.eliza/.elizadb
bun run dev  # Recreates database automatically

# Check database status
ls -la ./.eliza/.elizadb
````

### Port Conflicts

```bash
# Check if ports are in use
lsof -i :3001  # NUBI main server
lsof -i :5432  # PostgreSQL

# Change ports in .env
NUBI_SERVER_PORT=3002
```

### API Key Issues

```bash
# Test OpenAI connection
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"

# Check environment variables
echo $OPENAI_API_KEY | head -c 20
```

### Memory/Performance Issues

```bash
# Increase Node.js memory limit
export NODE_OPTIONS="--max-old-space-size=4096"

# Monitor memory usage
bun run dev --inspect
```
